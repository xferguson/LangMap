# ADR 0003: v1.2 Leaflet renderer + 250-language dataset + left side panel

## Status: Accepted (amends 0001 and 0002)

## Context

v1.1 shipped (ADR 0002): hand-rolled D3 SVG world map, 10-language seed
dataset, side panel on the right, 100/100 unit + 10/10 e2e green. The map
"shows the right shapes but doesn't feel like a map" (user feedback): no
graticule, no continents, no oceans, no geographic context. The seed
dataset of 10 languages is also too narrow for the app to be useful to
anyone who doesn't speak Spanish/English/French/German etc. v1.2 takes
three correlated swings:

1. Replace the renderer with Leaflet on a low-chroma tile basemap so the
   page reads as a real map.
2. Grow the dataset to 250 languages (floor 100), enforced by a
   build-time check that every official/recognized language for every
   country in the country GeoJSON is included.
3. Move the side panel from right to left.

The PRD v1.2 acceptance criteria (`docs/PRD.md` §v1.2, AC v1.2-1 through
v1.2-10) and the discovery brief (`docs/changes/v1.2-pivot.md`) capture
the kept/tweaked/removed boundary in detail. This ADR locks the technical
decisions and orders the tasks.

## Decision

1. **Renderer = `react-leaflet` v4 with imperative `setStyle`.**
   `<MapContainer>` mounts a Leaflet map. Two `<GeoJSON>` children: one
   for countries (assigned to a Leaflet pane `countries` at `zIndex
   400`), one for admin-1 (pane `admin1` at `zIndex 410`). The SVG
   renderer is forced (`preferCanvas={false}`) so each polygon exposes
   a DOM element. `onEachFeature` attaches the polygon's region code to
   the rendered SVG via
   `layer.on("add", () => layer.getElement()?.setAttribute("data-region", code))`.
   Highlighting is applied **imperatively** in a `useEffect` keyed on
   `coverageMap`: walk the layers, call `layer.setStyle({ fillOpacity })`
   on diff. Re-mounting `<GeoJSON>` on every state change would re-create
   ~400 polygons every keystroke; we don't.

2. **Tile provider = CartoDB Positron.** URL template
   `https://{s}.basemaps.cartocdn.com/light_all/{z}/{x}/{y}{r}.png`,
   subdomains `a` `b` `c` `d`, `maxZoom: 19`. Attribution is the Leaflet
   default attribution control with the string
   `&copy; <a href="https://www.openstreetmap.org/copyright">OpenStreetMap</a> contributors &copy; <a href="https://carto.com/attributions">CARTO</a>`.
   Free tier; no API key. Documented rate-limit posture: bursts >10
   req/s may be throttled — acceptable for personal/portfolio use.

3. **Layout = grid swap.** `src/styles.css`
   `.app { grid-template-columns: 360px 1fr }` (was `1fr 360px`). DOM
   order in `src/App.tsx` flips: `<aside>` becomes the first child of
   `<main>`, `<WorldMap>` the second. Side-panel width stays 360px.
   `html, body, #root { overflow: hidden }` and `.app { height: 100vh }`
   are unchanged.

4. **Picker = unchanged.** `src/components/LanguagePicker.tsx` body is
   not modified. A `<select>` with 250 `<option>`s + browser-native
   type-ahead is the right UX at this scale. v1.1-6 duplicate guard is
   preserved verbatim. No `<datalist>`, no custom combobox.

5. **Dataset growth = hand-authored must-include + curated regionals,
   build-enforced.** Source-of-truth is the union of:
   - `src/data/must-include.json` — hand-authored from the public
     "List of official languages by country and territory" reference.
     One record per `(countryIso, languageId, languageName)` pair for
     every official or de-facto recognized language by country.
     Committed to git; no live network fetch.
   - A curated regional-language list hand-authored in
     `src/data/regional-languages.ts` (~50 entries: Welsh, Basque,
     Catalan, Frisian, Sami, Tibetan, Quechua, Guaraní, etc.).

   The two unioned + deduped seed `src/data/languages.ts` to ~250
   entries. Each entry has speaker counts cited in a per-row
   `// source: <wikipedia or ethnologue link>` comment. `WORLD_POPULATION`
   in `src/lib/stats.ts` is unchanged at ~8.1B.

6. **Must-include enforcement = `scripts/check-coverage.ts` in
   `pretest` and `prebuild`.** Reads `must-include.json` and walks
   every `(country, language)` pair. Pair fails the check unless one
   of: (a) `country ∈ language.regions`, (b) `country ∈
   language.regions` for any sub-national child of `country` (i.e.
   `language.regions` contains `"ES-CT"` covers the requirement that
   Catalan be present for `ES`), or (c) the pair is documented in
   `src/data/must-include-exceptions.ts` with a one-line justification
   (e.g. politically disputed). Any unfixed failing pair fails CI.

7. **Admin-1 polygon coverage gap policy = warn-only at build, no-op
   at runtime.** `scripts/check-coverage.ts` also enumerates every
   `RegionRef` in `LANGUAGES` and checks for a matching feature in
   the admin-1 GeoJSON. Missing matches log a `WARN: language X
   references region Y but no polygon exists` and CI continues. The
   runtime renders nothing for the missing region. We do not fail the
   build (would block dataset PRs) and do not skip the language (its
   other regions still color correctly).

8. **Test contract migration.** `data-region` survives via the
   `onEachFeature` attribute attachment in §1. Unit tests for AC 1, 3,
   4, 5, 10, 11 stay in `tests/unit/WorldMap.test.tsx`, gated by a
   jsdom polyfill in `tests/setup.ts` that stubs
   `Element.prototype.getBoundingClientRect` to return non-zero
   dimensions (Leaflet expects a measurable container). Tests for
   v1.1-7 (white fill) and v1.1-8 (blue dominance) move to a new
   `tests/e2e/v1.2.spec.ts` because they require a real renderer to
   compute fill against tiles. New testids: the `<MapContainer>`
   receives `data-testid="world-map"` (replacing the old wrapper
   `<div>` that carried the same testid). A hidden
   `<div data-testid="map-coverage-state" data-coverage="<JSON>">`
   sits next to the map and serializes the current `coverageMap` so
   integration tests can assert region-opacity pairs without waiting
   for tile network or polygon paint.

9. **Performance = memo + RAF-debounce + diff-style.**
   - `coverageMap(state)` already cached behind a `useMemo` on
     `[state.languages, state.filter]`.
   - `<GeoJSON>` layers wrapped in `React.memo` keyed on dataset
     version (effectively constant).
   - `setStyle` is called only on polygons whose entry in the new
     `coverageMap` differs from the old one (diff-based update).
   - `LevelRangeSlider`'s `onChange` is RAF-debounced: a single
     `requestAnimationFrame` coalesces drag events before they reach
     `setState`.
   No virtualization is needed at 250 langs × ~400 polygons.

10. **GeoJSON pre-conversion at build time.** `world-atlas` ships
    TopoJSON; Leaflet doesn't read it natively. A new
    `scripts/build-geo.ts` runs once during `npm install` (postinstall)
    or manually, consumes `world-atlas/countries-110m.json`, converts
    via `topojson-client`, rewrites each feature's `id` to its ISO
    alpha-2 code (using a small lookup table), and writes
    `src/data/countries.geo.json`. The runtime imports this prebuilt
    GeoJSON directly. The `M49_TO_ISO` table from
    `src/components/WorldMap.tsx` migrates into the build script
    (consumes once, ships JSON keyed by ISO).

11. **Status of v1.1-3 (no scroll at 1280×800).** **Removed and
    replaced.** v1.2-9 specifies a softer floor: "no page-level scroll
    at desktop viewports ≥ 1024×768". Leaflet's container fills
    available height/width and never overflows; the side panel's
    internal list is the only scrollable region.

12. **Backwards-compat boundary.** Confirmed unchanged:
    `langmap.v1` localStorage key, payload schema
    (`{ languages: UserLanguage[], filter: { min: Level, max: Level } }`),
    `coverageMap` return type (`Map<RegionRef, number>`), `LANGUAGES`
    element shape, all four panel components' public props. A v1.1
    user reloading after deploying v1.2 keeps their saved languages
    intact.

## Consequences

- `package.json` gains `leaflet`, `react-leaflet`, `@types/leaflet`.
  `d3-geo`, `topojson-client`, `world-atlas` move to `devDependencies`
  (consumed only by the build script).
- New files: `scripts/build-geo.ts`, `scripts/check-coverage.ts`,
  `src/data/countries.geo.json` (generated), `src/data/admin1.geo.json`
  (generated and grown), `src/data/must-include.json`,
  `src/data/must-include-exceptions.ts`,
  `src/data/regional-languages.ts`, `tests/e2e/v1.2.spec.ts`,
  `tests/setup.ts` (jsdom polyfill).
- Dropped: the inline `M49_TO_ISO` table, the `geoNaturalEarth1`
  projection, the `<svg viewBox="0 0 960 500">` wrapper, the
  per-feature `<path>` JSX, `regionRefForCountry` /
  `regionRefForAdmin1` (replaced by GeoJSON `feature.properties.iso`).
- `src/data/languages.ts` grows from 10 to ~250 entries.
- `src/data/admin1.geo.json` grows from ~7 features to ~80.
- v1 + v1.1 data shapes (types, persistence, coverage, stats,
  duplicate guard) are unchanged.
- A v1.1 user upgrading hot-reloads into v1.2 without losing their
  saved languages.
- Tile load is a network dependency in e2e — Playwright config gains
  a route handler that mocks `*.basemaps.cartocdn.com` to a local
  blank PNG so CI doesn't hit the CDN.

## Task list

Tasks are ordered so QA can write tests for task N before task N+1
starts. Each task names the v1.2 AC(s) it covers.

1. **Add Leaflet dependencies and the jsdom polyfill.** `npm i
   leaflet react-leaflet @types/leaflet`. Move `d3-geo`,
   `topojson-client`, `world-atlas` to `devDependencies`. Add
   `tests/setup.ts` that stubs `Element.prototype.getBoundingClientRect`
   to return `{ width: 1024, height: 768, top: 0, left: 0, right: 1024,
   bottom: 768 }` and stubs `HTMLElement.prototype.clientWidth`/
   `clientHeight` similarly. Wire `tests/setup.ts` in `vitest.config.ts`
   via `setupFiles`. No production code yet. Reuse: nothing. Covers:
   v1.2-1 (test infrastructure for Leaflet under jsdom).

2. **Write `scripts/build-geo.ts`.** Consumes
   `node_modules/world-atlas/countries-110m.json`, converts via
   `topojson-client.feature(...)`, walks each feature, looks up its
   M49 id in an inline table (migrated from
   `src/components/WorldMap.tsx`), rewrites
   `feature.properties.iso = "<ISO>"`, drops `feature.id`, drops
   features whose M49 has no ISO mapping (the three disputed
   territories). Writes `src/data/countries.geo.json` (~600KB GeoJSON).
   Add to `package.json` scripts: `"build:geo": "tsx
   scripts/build-geo.ts"`. Reuse: the M49→ISO table from
   `WorldMap.tsx`. Covers: v1.2-1 (data prerequisite for the
   `<GeoJSON>` countries layer).

3. **Hand-author `src/data/must-include.json`.** From the public
   "List of official languages by country and territory" reference,
   produce an array of `{ countryIso: string, languageId: string,
   languageName: string }` records, one per official or
   de-facto-recognized language pair. Commit the JSON. No live
   network fetch; no SPARQL. The `languageId` matches the `id` field
   in `src/data/languages.ts` so task 4's coverage check is a simple
   lookup. Reuse: nothing. Covers: prerequisite for v1.2-4.

4. **Write `scripts/check-coverage.ts`.** Consumes
   `src/data/languages.ts`, `src/data/must-include.json`,
   `src/data/countries.geo.json`, and `src/data/admin1.geo.json`.
   For every `(countryIso, languageId)` in `must-include.json`,
   find the matching `Language` entry in `LANGUAGES` by `id` and
   verify that `language.regions` includes either the country code
   directly or any of its sub-national codes from
   `admin1.geo.json`. Failing pairs not listed in
   `src/data/must-include-exceptions.ts` cause `process.exit(1)`.
   Also walk every `RegionRef` in `LANGUAGES.flatMap(l => l.regions)`
   and warn (no exit) on any that doesn't match a feature in the
   country or admin-1 GeoJSON. Wire into `package.json` `pretest`
   and `prebuild`. Reuse: `src/types.ts`. Covers: v1.2-3, v1.2-4.

5. **Grow `src/data/admin1.geo.json` from ~7 to ~80 features.**
   Source: Natural Earth `ne_50m_admin_1_states_provinces`, filtered
   to the subdivisions referenced by the curated regional-language
   list (task 6) plus the v1.1 set (Catalonia, Balearics, Valencia,
   Quebec, Wales, Hong Kong, Macau, Andorra). Each feature's
   `properties.iso` is set to its ISO 3166-2 code. Document the
   `mapshaper` command in `README.md`. Reuse: existing
   `admin1-subset.json` as the starting set. Covers: v1.2-3, v1.2-6.

6. **Grow `src/data/languages.ts` to ~250 entries.** Two passes:
   (a) `regional-languages.ts` (curated): Welsh, Basque, Catalan,
   Galician, Frisian, Sami (North/Lule/South), Tibetan, Uyghur,
   Quechua, Guaraní, Aymara, Maori, Hawaiian, Navajo, Cherokee,
   Inuktitut, Faroese, Greenlandic, Romansh, Luxembourgish, Sorbian,
   Kashubian, Sardinian, Sicilian, Friulian, Ladin, Occitan, Breton,
   Corsican, Manx, Cornish, Scottish Gaelic, Irish, Yiddish, Ladino,
   Aramaic, Coptic, Tigrinya, Amharic, Oromo, Somali, Hausa, Yoruba,
   Igbo, Fulani, Wolof, Akan, Zulu, Xhosa, Tsonga, Sotho, Setswana,
   Shona, Swahili, Kinyarwanda, Lingala, Kikongo (selection — full
   curated list locked at task time). (b) Merge with the languages
   referenced by `must-include.json` (deduped by `id`, sorted
   alphabetically by `name`). Each entry: `{ id, name, endonym?,
   speakers: { l1, l2 }, regions[] }` — same shape as v1, no schema
   change. `id` is the canonical identifier (typically ISO 639-3
   alpha-3 like `"spa"`, `"cat"`, `"yue"`, or BCP-47 where ISO 639-3
   doesn't disambiguate). Speaker counts cite Ethnologue or Wikipedia
   per row. Reuse: existing 10-language seed becomes the test
   fixture. Covers: v1.2-3, v1.2-4.

7. **Create `src/data/languages.fixture.ts` for stable test
   inputs.** Copy the v1.1 10-language seed verbatim into a fixture
   file so unit tests have a stable small input independent of the
   growing dataset. Repoint all v1 / v1.1 unit tests that import
   `LANGUAGES` from `src/data/languages.ts` to import from the
   fixture instead. The growing `languages.ts` is consumed only by
   the production app and the e2e specs that exercise the real
   user-facing list. Type unchanged — no `wikidataId` field, just
   the existing `{ id, name, endonym?, speakers, regions }` shape.
   Reuse: existing 10-language list. Covers: prerequisite for tasks
   4 and 6.

8. **Rewrite `src/components/WorldMap.tsx` to use Leaflet.** Body is
   replaced wholesale. New body:
   - `<MapContainer center={[20, 0]} zoom={2} minZoom={2} maxZoom={6}
     scrollWheelZoom data-testid="world-map" style={{ height: "100vh",
     width: "100%" }}>`.
   - `<TileLayer url="https://{s}.basemaps.cartocdn.com/light_all/{z}/{x}/{y}{r}.png"
     subdomains="abcd" attribution="&copy; OSM contributors &copy;
     CARTO" maxZoom={19} />`.
   - Two memoized `<GeoJSON>` children: one for countries (loaded
     from `src/data/countries.geo.json`), one for admin-1 (from
     `src/data/admin1.geo.json`). Each receives an `onEachFeature`
     that attaches `data-region` after layer-add and a `style`
     callback returning `{ fillColor: "#3b82f6", fillOpacity: 0,
     weight: 0.4, color: "#9ca3af" }` initially.
   - A `useEffect` keyed on `coverageMap` walks the two layer groups

   - A hidden `<div data-testid="map-coverage-state"

   - The component's outer wrapper retains `data-testid="world-map"`

   Reuse: `src/lib/coverage.ts` (`coverageMap`), `src/types.ts`.

## Follow-ups (recorded for the final architectural check)

- **Dataset size below target.** `LANGUAGES.length === 139` after the
  v1.2 cycle, below the 250-entry target in §5. The 100-entry floor is
  cleared. v1.3 task: continue regional/long-tail growth (Indo-Aryan
  minor, Bantu, Pacific, Caucasian, indigenous Americas) toward the
  full 250.
- **Must-include enumeration partial.** `src/data/must-include.json`
  ships ~50 (61 actual) `(country, language)` pairs covering the
  highest-impact countries (US, GB, CA, ES, MX, DE, FR, BR, RU, CN,
  JP, IN, etc.) and the multi-language ones that exercise the
  enforcement meaningfully (BE, CH, LU, IL, NZ). v1.3 task: expand to
  the full ~150-pair enumeration from the public "List of official
  languages by country and territory" reference for global coverage.
- **jsdom Leaflet polyfill.** `tests/setup.ts` adds an
  `SVGSVGElement.prototype.createSVGRect` polyfill (alongside the
  bounding-rect / clientWidth / clientHeight stubs documented in
  §task 1). The implementer added it in v1.2 cycle pass 1: without
  it Leaflet's `Browser.svg` probe decides SVG is unsupported, `L.svg()`
  returns null, and vector layers silently fail to render under the
  unit-test runner. QA should formally adopt this addition into the
  testid/setup contract; no further runtime change is needed.
